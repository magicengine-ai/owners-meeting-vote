# 微信云托管关键配置说明

## 🎯 核心特性（来自官方文档）

### 1. 端口配置
**微信云托管默认端口：8080**

```python
# config.py 已自动适配
PORT: int = int(os.getenv("PORT", "8080"))
```

> ⚠️ 注意：不是 5000 也不是 3000，必须是 8080！

---

### 2. 微信原生鉴权（重要！）

微信云托管最大的优势：**自动获取微信用户信息，无需维护 access_token**

#### 请求头自动注入
```
X-WX-OPENID: 用户的 OpenID
X-WX-UNIONID: 用户的 UnionID（如果绑定）
```

#### 代码示例
```python
from fastapi import Request

@app.get("/api/user/info")
async def get_user_info(request: Request):
    # 直接从请求头获取 OpenID，无需解密
    openid = request.headers.get("X-WX-OPENID")
    unionid = request.headers.get("X-WX-UNIONID")
    
    return {"openid": openid, "unionid": unionid}
```

### 3. 免鉴权调用微信接口

无需维护 access_token，直接调用微信开放接口：

```python
import httpx

# 传统方式（需要维护 access_token）
# access_token = get_access_token()
# url = f"https://api.weixin.qq.com/xxx?access_token={access_token}"

# 云托管方式（免鉴权）
async with httpx.AsyncClient() as client:
    response = await client.get(
        "https://api.weixin.qq.com/xxx",
        headers={
            "X-WX-APPID": settings.WECHAT_APP_ID,
            "X-WX-SECRET": settings.WECHAT_APP_SECRET
        }
    )
```

### 4. 数据库配置

#### 自动注入的环境变量
```bash
# MySQL（云托管提供）
TENCENTCLOUD_MYSQL_HOST
TENCENTCLOUD_MYSQL_PORT
TENCENTCLOUD_MYSQL_DATABASE
TENCENTCLOUD_MYSQL_USERNAME
TENCENTCLOUD_MYSQL_PASSWORD

# Redis（可选）
TENCENTCLOUD_REDIS_HOST
TENCENTCLOUD_REDIS_PORT
TENCENTCLOUD_REDIS_PASSWORD
```

#### 数据库连接字符串
```python
# db.py 已自动适配
if DB_TYPE == "mysql":
    SQLALCHEMY_DATABASE_URL = (
        f"mysql+pymysql://{DB_USER}:{DB_PASSWORD}"
        f"@{DB_HOST}:{DB_PORT}/{DB_NAME}"
        f"?charset=utf8mb4"
    )
```

### 5. 三种部署方式

#### 方式一：流水线部署（推荐）
```
GitHub/GitLab 代码仓库 
    → 自动构建 
    → 自动部署
```

**优点：**
- 代码提交后自动部署
- 支持回滚到历史版本
- 适合团队协作

#### 方式二：上传代码包
```
本地代码 
    → 压缩成 ZIP 
    → 上传到云托管控制台
```

**优点：**
- 简单直接
- 适合快速测试

#### 方式三：上传镜像
```
本地构建 Docker 镜像 
    → 推送到镜像仓库 
    → 云托管拉取部署
```

**优点：**
- 完全控制构建环境
- 适合复杂依赖

### 6. 本地调试

微信云托管提供本地调试工具：

```bash

npm install -g @cloudbase/cli

# 本地构建镜像
cloudbase run --port 8080

cloudbase invoke --openid "模拟的 OpenID"
```

### 8. 扩缩容配置

#### 推荐配置
```yaml
最小实例数：0      # 夜间无流量时缩容到 0，节省成本
最大实例数：10     # 根据流量自动扩容
常驻实例：0-2      # 可选，避免冷启动
```

#### 冷启动优化
如果担心冷启动慢（5-10 秒），可以：
1. 设置最小实例数为 1-2
2. 使用「常驻实例」功能
3. 优化代码启动速度

## 🔐 安全建议

1. **不要将密钥写入代码**
   - 所有密钥使用环境变量
   - 使用云托管的「环境变量」功能

2. **开启 VPC 内网访问**
   - 数据库只在 VPC 内访问
   - 不开放公网访问

3. **配置 CORS**
   ```python
   # 生产环境限制域名
   allow_origins=["https://your-miniprogram.com"]
   ```

4. **开启访问日志**
   - 记录所有请求
   - 便于审计和排查问题
