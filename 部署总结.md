## 🚀 快速启动（3 步）

### 方式一：一键启动（推荐）

```bash
# 双击运行
quick-start.bat

# 或使用 PowerShell
.\deploy.ps1 -Docker
```

### 方式二：手动启动

```bash
# 1. 进入 docker 目录
cd docker

# 2. 复制环境配置
copy .env.example .env

# 3. 编辑 .env 文件，填入配置
notepad .env

# 4. 启动服务
docker-compose up -d

# 5. 查看状态
docker-compose ps
```

## 📋 必需配置项

在 `docker/.env` 文件中配置以下关键参数：

### 1. 数据库密码（必需）
```bash
DB_PASSWORD=your_strong_password
```

### 2. JWT 密钥（必需）
```bash
# 生成命令
openssl rand -hex 32
# 或
python -c "import secrets; print(secrets.token_urlsafe(32))"

# 填入生成的密钥
JWT_SECRET_KEY=生成的密钥
```

### 3. 微信小程序配置（开发可跳过）
```bash
WECHAT_APP_ID=wx0000000000000000
WECHAT_APP_SECRET=your_secret
```

### 4. 百度 OCR 配置（开发可跳过）
```bash
OCR_APP_ID=your_app_id
OCR_API_KEY=your_api_key
OCR_SECRET_KEY=your_secret_key
```

## 🔧 常用命令

### Docker 管理
```bash
# 查看服务状态
docker-compose ps

# 查看所有日志
docker-compose logs -f

# 停止服务
docker-compose down

# 重启服务
docker-compose restart

# 重新构建
docker-compose build --no-cache

# 清理资源
docker-compose down -v
```

### 数据库操作
```bash
# 进入数据库
docker-compose exec db psql -U postgres -d vote_db

# 运行迁移
docker-compose exec backend alembic upgrade head

# 回滚迁移
docker-compose exec backend alembic downgrade -1

# 备份数据库
docker-compose exec db pg_dump -U postgres vote_db > backup.sql

# 恢复数据库
docker-compose exec -T db psql -U postgres vote_db < backup.sql
```

### 后端操作
```bash
# 进入后端容器
docker-compose exec backend bash

# 创建管理员
docker-compose exec backend python scripts/create_admin.py

# 查看 Python 版本
docker-compose exec backend python --version

# 安装依赖
docker-compose exec backend pip install -r requirements.txt
```

## 🛡️ 安全建议

### 1. 修改默认密码
- ✅ 数据库密码使用强密码
- ✅ JWT_SECRET_KEY 使用随机生成的密钥
- ✅ 定期更换密钥

### 2. 网络隔离
- ✅ Docker 网络隔离（已配置）
- ✅ 数据库不暴露到公网
- ✅ 使用 Nginx 反向代理

### 3. 数据备份
```bash
# 定期备份数据库
docker-compose exec db pg_dump -U postgres vote_db > backup_$(date +%Y%m%d).sql

# 备份 Redis 数据
docker-compose exec redis redis-cli BGSAVE
```
