## 📝 Git 工作流

### 分支策略

```
main          - 主分支（稳定版本）
  └── develop    - 开发分支
       └── feature/xxx - 功能分支
```

### 提交规范

```
<type>: <subject>

<body>
```

**Type 类型：**
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

**示例：**
```bash
git commit -m "feat: 实现微信登录功能

- 对接微信 OAuth2.0
- 实现 JWT Token 生成
- 添加用户信息存储"
```
