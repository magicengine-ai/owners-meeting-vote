## ⚠️ 已踩过的坑

### 1. 中文编码问题

**问题：** 文件保存时使用了错误的编码（UTF-8 BOM 或 GBK），导致中文字符显示为乱码

**错误示例：**
```javascript
// ❌ 乱码
console.log('涓氫富澶т細鎶曠エ灏忕▼搴忓惎？');
verifyText: '鏈よ瘉？',
title: '鐧诲綍宸茶繃鏈？',
```

**正确做法：**
```javascript
// ✅ UTF-8 无 BOM 编码
console.log('业主大会投票小程序已启动');
verifyText: '未认证',
title: '登录已过期',
```

**预防措施：**
- ✅ 使用 VSCode 编辑器，右下角确认编码为 `UTF-8`
- ✅ 保存时选择 `Save with Encoding → UTF-8`
- ✅ 避免复制粘贴来源不明的代码

### 2. Require 路径层数错误

**问题：** 手动计算相对路径时层数错误，导致模块加载失败

**错误示例：**
```javascript
// pages/profile/profile.js
const { get } = require('../../../utils/request.js')  // ❌ 3 层，应该是 2 层

// pages/vote/list/list.js
const { get } = require('../../../../utils/request.js')  // ❌ 4 层，应该是 3 层
```

**正确路径：**
```javascript
// pages/profile/profile.js (2 层)
const { get } = require('../../utils/request.js')  // ✅

// pages/vote/list/list.js (3 层)
const { get } = require('../../../utils/request.js')  // ✅

// pages/admin/verify-list/verify-list.js (3 层)
const { get } = require('../../../utils/request.js')  // ✅
```

**路径计算规则：**
```
pages/profile/profile.js     → 2 层 → ../../utils/request.js
pages/vote/list/list.js      → 3 层 → ../../../utils/request.js
pages/admin/verify-list.js   → 3 层 → ../../../utils/request.js
```

**预防措施：**
- ✅ 数目录层数：从当前文件所在目录往上数到 `pages`，再加 1 层到 `frontend`，再加 `utils`
- ✅ 使用 IDE 的路径自动补全功能
- ✅ 提交前测试所有 require 语句

### 3. WXML 语法限制

**问题：** 微信小程序 WXML 不支持管道语法和方法调用

**错误示例：**
```xml
<!-- ❌ 不支持管道语法 -->
<text>{{userInfo.openid | substring: 0:8}}</text>

<!-- ❌ 不支持方法调用 -->
<text>{{(item.percentage * 100).toFixed(1)}}%</text>
```

**正确写法：**
```xml
<!-- ✅ 使用预计算的变量 -->
<text>{{userInfo.openidSubstring}}</text>
<text>{{item.percentageText}}%</text>
```

**对应的 JS：**
```javascript
// 在 JS 中计算好值
const userInfo = {
  openidSubstring: res.openid ? res.openid.substring(0, 8) : '',
  // ...
}

const voteResults = res.results.map(item => ({
  ...item,
  percentageText: (item.percentage * 100).toFixed(1)
}))
```

**预防措施：**
- ✅ WXML 只负责展示，不做计算
- ✅ 所有计算在 JS 的 `loadXXX()` 方法中完成
- ✅ 使用 `setData()` 传入计算后的值

### 4. 项目目录混乱

**问题：** 创建了多个相似的项目文件夹，导致代码不同步

**错误示例：**
```
agents/engineer/projects/
├── 业主投票小程序/          ❌ 旧项目（已删除）
└── 业主大会投票小程序/      ✅ 正确项目
```

**预防措施：**
- ✅ 一个项目只保留一个目录
- ✅ 删除旧目录前先确认新目录代码完整
- ✅ 使用 Git 管理，不要手动复制目录

### 5. 微信开发者工具缓存

**问题：** 代码已修复，但编译仍报旧错误

**症状：**
- 文件已修改并保存
- Git 已提交并推送
- 微信开发者工具仍报旧错误

**解决方案：**
```
1. 工具 → 清除缓存 → 清除全部缓存 (Ctrl+Shift+Delete)
2. 关闭微信开发者工具
3. 重新打开项目
4. 重新编译
```

**预防措施：**
- ✅ 每次修复后必须清除缓存
- ✅ 如果问题仍存在，关闭工具重新打开
- ✅ 不要相信"热更新"，手动触发编译

### 6. 大文件提交

**问题：** 提交了大文件（如 `vote-api.tar` 222MB），导致 Git 推送失败

**错误示例：**
```
remote: error: File vote-api.tar is 222.14 MB
remote: error: GH001: Large files detected
```

**解决方案：**
```bash
# 从 Git 历史中删除大文件
git reset --hard HEAD~1
# 添加到 .gitignore
echo "vote-api.tar" >> .gitignore
# 重新提交
git add .gitignore
git commit -m "chore: 添加大文件到 gitignore"
```

**预防措施：**
- ✅ 在 `.gitignore` 中添加：
  ```
  *.tar
  *.zip
  node_modules/
  .DS_Store
  ```
- ✅ 提交前检查文件大小：`git status`
- ✅ 使用 Git LFS 管理大文件（如必须提交）
